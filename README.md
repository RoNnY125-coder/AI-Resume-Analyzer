# 📄 AI Resume Analyzer

> **Multi-dimensional resume scoring powered by TF-IDF & NLP**

Upload your PDF resume, paste a job description, and get an instant AI-powered analysis with a score, letter grade, and actionable feedback — all in seconds.

---

## 🚀 Live Demo

Deploy your own instance on Vercel with one click (see [Deployment](#deployment)).

---

## ✨ Features

- 📊 **6-Dimension Scoring** — Keyword Match, Section Structure, Impact & Metrics, Skills Depth, Formatting Quality, Action Verb Usage
- 🎯 **Overall Score (0–100)** with letter grade (A+ to F)
- 🔑 **Missing Keywords** — Top 10 keywords from the job description absent in your resume
- 💡 **Suggestions & Strengths** — Specific, actionable feedback personalized to your resume
- 📑 **Section Analysis** — Detects which standard resume sections are present
- 📎 **Drag & Drop Upload** — Accepts text-based PDF resumes up to 10 MB
- ⚡ **Fast** — Results in 1–3 seconds; no login required

---

## 🖼️ How It Works

```
User uploads PDF + pastes job description
        ↓
FastAPI extracts text from PDF (pdfplumber)
        ↓
NLP preprocessing (tokenize, lowercase, stopword removal)
        ↓
6 scoring dimensions computed in parallel
        ↓
Overall score = weighted average of 6 dimensions
        ↓
Missing keywords + suggestions + strengths generated
        ↓
JSON response rendered in the browser UI
```

---

## 📐 Scoring Dimensions

| Dimension | Weight | Method |
|---|---|---|
| Keyword Match | 25% | TF-IDF cosine similarity |
| Section Structure | 20% | Keyword-based section detection |
| Impact & Metrics | 20% | Regex (%, $, numbers, impact phrases) |
| Skills Depth | 15% | Curated skills vocabulary matching |
| Formatting Quality | 10% | Word count, contact info, capitalization |
| Action Verb Usage | 10% | Curated action verb vocabulary |

### Grade Scale

| Grade | Score |
|---|---|
| A+ | 90–100 |
| A | 85–89 |
| B+ | 80–84 |
| B | 70–79 |
| C+ | 60–69 |
| C | 50–59 |
| D | 40–49 |
| F | < 40 |

## 📡 API Reference

### `POST /analyze`

Analyzes a resume against a job description.

**Request** — `multipart/form-data`

| Field | Type | Description |
|---|---|---|
| `resume` | File | PDF resume (text-based, max 10 MB) |
| `job_desc` | String | Full job description text (min 20 chars) |

**Response** — `200 OK`

```json
{
  "overall_score": 72.45,
  "grade": "B",
  "dimension_scores": {
    "keyword_match": 65.3,
    "section_structure": 80.0,
    "impact_metrics": 55.0,
    "skills_depth": 70.5,
    "formatting": 85.0,
    "action_verbs": 60.0
  },
  "keywords_missing": ["kubernetes", "terraform", "docker"],
  "suggestions": ["Add more quantifiable achievements"],
  "strengths": ["Strong technical skills alignment"],
  "section_analysis": {
    "contact": true,
    "education": true,
    "experience": true,
    "skills": true,
    "projects": false,
    "summary": false
  },
  "contact_info": {
    "email": true,
    "phone": true,
    "linkedin": false,
    "github": true
  }
}
```

### `GET /health`

## 💡 Tips for Best Results

- Use a **text-based PDF** — scanned image PDFs cannot be parsed
- Paste the **complete job description**, not just the title
- A score above **70** generally indicates a strong application match
- Use the **missing keywords** list to tailor your resume for each role
- Aim for **300–800 words** in your resume for best formatting score

## ⚠️ Limitations

- Optimized for English-language resumes and job descriptions
- Skills vocabulary is weighted toward tech/engineering roles
- Does not evaluate resume visual design or layout

## Bug Fixes (v3.0.0)

| Bug | Root Cause | Fix |
|-----|-----------|-----|
| `match_score = 0` | TF-IDF fitted on one doc, no shared vocabulary | Fit vectorizer on BOTH docs together |
| Bad keywords ("string") | No quality filter on extracted tokens | Added `isalpha()` + min-length filter |
| PDF extraction failures | PyPDF2 didn't handle None page text | pdfplumber with per-page None guard |
| Smart quotes breaking tokenisation | PDF unicode not normalised | NFKD unicode normalisation before cleaning |
